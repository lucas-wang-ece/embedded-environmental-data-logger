# Learning Notes

## V1: LED Blink

### Concepts
- `setup()` runs once when the board starts or resets.
- `loop()` runs repeatedly after `setup()`.
- `pinMode(pin, OUTPUT)` configures a pin as output.
- `digitalWrite(pin, HIGH)` sets the pin to a high voltage level.
- `digitalWrite(pin, LOW)` sets the pin to a low voltage level.
- `delay(1000)` pauses the program for 1000 milliseconds.

## V2: Serial Monitor Output

### Concepts
- `Serial.begin(9600)` starts serial communication at 9600 baud.
- The Serial Monitor baud rate must match the value in `Serial.begin()`.
- `Serial.print()` prints without moving to a new line.
- `Serial.println()` prints and then moves to a new line.
- Serial output is useful for debugging firmware runtime behavior.

## V3: DHT11 Sensor Reading

### Concepts
- The DHT11 sensor measures temperature and humidity.
- The sensor has three pins: power, data output, and ground.
- The data output pin is connected to Arduino digital pin 2.
- `dht.begin()` initializes the DHT11 sensor.
- `dht.readTemperature()` reads the temperature in Celsius.
- `dht.readHumidity()` reads the humidity percentage.
- `isnan()` is used to check whether the sensor reading failed.

## V4: Threshold-Based Alert

### Concepts
- Sensor readings can be used as inputs for control logic.
- A temperature threshold is defined using `TEMPERATURE_THRESHOLD_C`.
- `if / else` logic determines whether the alert should be on or off.
- `digitalWrite(ALERT_LED_PIN, HIGH)` turns the alert LED on.
- `digitalWrite(ALERT_LED_PIN, LOW)` turns the alert LED off.
- This version connects sensor input, decision-making logic, and hardware output.

## V5: Timer-Driven Sampling

### Concepts
- `delay()` blocks the main program while waiting.
- Timer-driven sampling allows the system to request sensor readings at fixed time intervals.
- A timer interrupt should stay short and should not perform slow operations.
- The interrupt service routine only sets a `sampleRequested` flag.
- The main `loop()` checks the flag and performs the actual sensor reading.
- `volatile` is used for variables shared between the interrupt service routine and the main loop.
- This structure makes the firmware more responsive and easier to extend.

## V6: SD Card CSV Logging

### Concepts
- The microSD card module communicates with the Arduino through the SPI interface.
- SPI uses multiple signal lines: CS, SCK, MOSI, and MISO.
- `SD.begin(SD_CS_PIN)` initializes communication with the microSD card.
- The CS pin is used to select the SD card module during SPI communication.
- `SD.open()` opens or creates a file on the microSD card.
- `FILE_WRITE` allows the Arduino to append new data to the file.
- `logFile.print()` writes data without moving to a new line.
- `logFile.println()` writes data and then moves to a new line.
- `logFile.close()` saves and closes the file after writing.
- CSV files store data in comma-separated format and can be opened later in spreadsheet software.
- `millis()` records the elapsed time since the Arduino started running.
- The final V6 version combines timer-driven sampling with SD card logging.

### Wiring
- SD Module CS    -> Arduino D10
- SD Module SCK   -> Arduino D13
- SD Module MOSI  -> Arduino D11
- SD Module MISO  -> Arduino D12
- SD Module VCC   -> Arduino 5V
- SD Module GND   -> Arduino GND

### Summary
V6 added SPI-based microSD card logging to the environmental data logger. The system reads temperature and humidity data from the DHT11 sensor at timer-driven intervals, updates the LED alert based on a temperature threshold, prints diagnostic output through Serial Monitor, and saves structured sensor readings to `sensor.csv` on the microSD card.

## Development Environment

### Concepts
- Compile means checking and converting the code into machine-readable instructions.
- Upload means transferring the compiled firmware to the microcontroller.
- COM port is the communication port used by the computer to talk to the Arduino board.
